# Инструкция: подключение компьютера к производственным папкам Paper Planes

Дата: 2026-06-27

## Цель

Настроить компьютер так, чтобы сотрудник работал с проектными файлами через корпоративный Google Drive и сохранял новые материалы в правильную производственную папку.

Теперь правило шире, чем старое правило по РГ2. Нас интересует вся производственная зона на общем диске Paper Planes.

## Две целевые папки

Есть две папки, где лежат проекты производства:

1. `70.1.2 РГ2`
   https://drive.google.com/drive/folders/18OdPDOhWblfsSOhyKw5z2kyGRM2nxGpu

2. `Реализация` внутри папки РГ1
   https://drive.google.com/drive/folders/1GFqllhIRkM_j9GR7Vxj1_Bh4uMCHYvLi

Все файлы, которые сотрудник создаёт по проекту, кладутся в папку этого проекта внутри одной из этих двух зон.

## Правило выбора папки

1. Сначала ищем целевой проект в `70.1.2 РГ2`.
2. Потом ищем тот же проект в `Реализация` внутри РГ1.
3. Если проект найден только в одной зоне, новые файлы сохраняем туда.
4. Если проект найден в обеих зонах, останавливаемся и уточняем у Натальи, какая папка каноническая.
5. Если проект не найден ни в одной зоне, останавливаемся и уточняем у Натальи, куда его заводить.

Папку проекта нельзя создавать молча.

## Настройка Google Drive for desktop

1. Установить Google Drive for desktop.
2. Войти под корпоративным аккаунтом Paper Planes.
3. Открыть Google Drive в Finder.
4. Проверить, что доступен общий диск `Paper Planes`.
5. Открыть путь:

```text
~/Library/CloudStorage/GoogleDrive-<аккаунт>/Shared drives/Paper Planes/4. Производство/
```

На русской локали Google Drive может показывать `Общие диски` вместо `Shared drives`. В Finder название может отличаться, в терминале чаще остаётся `Shared drives`.

## Что должно быть доступно локально

Минимум:

```text
Paper Planes/4. Производство/.../70.1.2 РГ2/
Paper Planes/4. Производство/.../Реализация/
```

Для активных проектов лучше включить локальную доступность папки проекта:

1. Finder.
2. Найти папку проекта.
3. Правый клик.
4. Google Drive.
5. `Available offline` / `Доступно офлайн`.

Это снижает риск, что файл виден в вебе, но ещё не подтянулся локально.

## Проверка перед работой

В комплекте есть скрипт:

```text
pp_production_drive_check.sh
```

Он ничего не меняет на компьютере. Только проверяет:

- запущен ли Google Drive for desktop;
- есть ли локальные Google Drive-корни;
- виден ли общий диск `Paper Planes`;
- найдена ли папка `4. Производство`;
- найдены ли целевые зоны РГ2 и реализации;
- где лежит конкретный проект.

Запуск:

```bash
cd /папка/где/лежит/скрипт
chmod +x pp_production_drive_check.sh
./pp_production_drive_check.sh
```

Проверка конкретного проекта:

```bash
./pp_production_drive_check.sh "Гартенн"
```

## Как читать результат сканера

`OK` — найдено и выглядит рабочим.

`WARN` — есть подозрение: например, Google Drive не запущен или целевая папка не найдена по имени.

`FAIL` — базовая настройка не работает: нет локального Google Drive или не видна папка производства.

`FOUND` — найден кандидат на папку проекта.

`STOP` — нельзя сохранять файл, пока не уточнили место.

## Контрольное правило для сотрудников

Перед сохранением нового файла сотрудник должен ответить на один вопрос:

```text
Где уже лежит папка этого проекта: в РГ2 или в Реализации?
```

Файл сохраняется туда же.

## Что делать на компьютере, где уже что-то настроено

1. Запустить сканер без названия проекта.
2. Проверить, видна ли папка `4. Производство`.
3. Запустить сканер с названием проекта.
4. Если сканер нашёл один путь, использовать его.
5. Если сканер нашёл несколько путей, ничего не переносить и не удалять.
6. Если сканер ничего не нашёл, открыть две ссылки из раздела «Две целевые папки» в браузере и проверить доступы.

## Что не делать

- Не сохранять проектные файлы в `Downloads`, на рабочий стол или в личный `My Drive`.
- Не создавать новую папку проекта без подтверждения.
- Не переносить файлы между РГ2 и Реализацией без отдельной команды.
- Не ориентироваться только на тип проекта. Ориентируемся на фактическое место, где уже лежит папка проекта.
